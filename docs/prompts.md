# Prompts

Este documento almacena los prompts reutilizables utilizados durante el desarrollo del proyecto.

---

# Revisión de Diseño

Analiza el siguiente componente desde el punto de vista de un diseñador UX/UI Senior.

Evalúa:

- Coherencia visual con el resto del sitio.
- Elegancia.
- Minimalismo.
- Jerarquía visual.
- Espaciados.
- Tipografía.
- Uso del color.
- Contraste.
- Responsive (Mobile y Desktop).
- Accesibilidad visual.
- Microinteracciones.
- Animaciones.
- Consistencia con la identidad del proyecto.

No busques únicamente errores técnicos.

Critica también las decisiones de diseño.

Si alguna decisión rompe la coherencia visual del sitio explícalo y propone alternativas.

No seas complaciente.

---

# Revisión de Código

Actúa como un Senior Frontend Engineer especializado en Next.js.

Analiza el siguiente código considerando:

- SEO.
- Accesibilidad.
- Rendimiento.
- Buenas prácticas.
- Modularización.
- Legibilidad.
- Escalabilidad.
- Arquitectura.
- Tipado TypeScript.
- Seguridad.
- Uso correcto de React.
- Uso correcto de Next.js.

No reescribas el componente completo.

Primero analiza.

Luego explica los problemas encontrados.

Finalmente propone mejoras justificando cada una.

---

# Generador de Prompts para Copilot

Necesito que generes un prompt para GitHub Copilot.

El prompt debe:

- Ser extremadamente claro.
- No asumir contexto oculto.
- Especificar tecnologías.
- Explicar restricciones.
- Describir exactamente el resultado esperado.

Después de generar el prompt NO desarrolles la solución.

Solo entrega el prompt.

Posteriormente analizaremos la respuesta generada por Copilot.

---

# Generación de Código

Genera código siguiendo estrictamente el stack del proyecto.

Tecnologías:

- Next.js
- React
- TypeScript
- CSS Modules
- Node.js
- Supabase
- SQL
- n8n

Restricciones:

- Mobile First.
- Server Components por defecto.
- Client Components solo cuando sean necesarios.
- Componentes pequeños.
- Una responsabilidad por componente.
- Sin librerías innecesarias.
- Código limpio.
- SEO.
- Accesibilidad.
- Buen rendimiento.

Antes de escribir código explica brevemente la estrategia.

---

# Documentación

Genera o actualiza documentación técnica del proyecto.

Puede tratarse de:

- README.md
- documentación técnica
- documentación funcional
- archivos dentro de /docs

La documentación debe ser clara, profesional y mantenerse consistente con el resto del proyecto.

Evita duplicar información existente.

Si detectas inconsistencias propón mejoras.
