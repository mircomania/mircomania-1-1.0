# Auditoría Completa del Proyecto (Copilot)

Actúa como un Staff Frontend Engineer con experiencia en aplicaciones de producción.

Analiza el proyecto completo y busca oportunidades reales de mejora.

No te limites al archivo abierto; considera la arquitectura, la organización del código y la coherencia general.

Evalúa:

## Arquitectura

- Organización de carpetas.
- Separación por responsabilidades.
- Escalabilidad.
- Acoplamiento entre componentes.
- Reutilización de código.

## React

- Renderizados innecesarios.
- Estados redundantes.
- Hooks reutilizables.
- Memoización cuando realmente aporte valor.
- Posibles bugs.

## Next.js

- Uso correcto de Server Components.
- Uso correcto de Client Components.
- Carga de datos.
- SSR.
- Optimización de imágenes.
- Metadata.
- Rutas.

## TypeScript

- Tipado.
- Tipos repetidos.
- Posibles mejoras.

## CSS

- CSS repetido.
- Variables reutilizables.
- Responsive.
- Mobile First.
- Mantenibilidad.

## SEO

- HTML semántico.
- Jerarquía de headings.
- Metadata.
- Alt.
- Lighthouse.

## Accesibilidad

- Navegación por teclado.
- Screen readers.
- aria.
- Contraste.
- Focus.

## Rendimiento

- Código innecesario.
- Re-renderizados.
- Bundle.
- Lazy Loading.
- Animaciones costosas.
- Optimización para dispositivos móviles.

## Seguridad

- Variables de entorno.
- Acceso a APIs.
- Exposición de datos.
- Buenas prácticas.

## Calidad del código

- Código duplicado.
- Funciones demasiado largas.
- Componentes demasiado grandes.
- Posibles refactorizaciones.

---

No propongas cambios únicamente por gusto.

Justifica cada sugerencia indicando:

- qué problema resuelve,
- por qué mejora el proyecto,
- cuál sería el beneficio real.

Prioriza las mejoras según impacto:

🔴 Alto impacto

🟡 Impacto medio

🟢 Bajo impacto

No seas complaciente. Critica el proyecto como si estuvieras realizando una revisión técnica para aprobar una aplicación antes de salir a producción.
